# GitHub Action لمعالجة ورفع ملفات APKs/APKM/APKX (مع توليد الميتاداتا و DM)

هذا الـ GitHub Action المطور يقوم بتحميل ملفات حزم التطبيقات، فك ضغطها، اختيار ملفات محددة، ثم **توليد ملفات التحقق والميتاداتا وملفات تحسين الأداء (base.dm) تلقائياً** قبل إعادة ضغطها ورفعها.

## الميزات المضافة:
- **توليد `base.dm` (Dex Metadata)**: ملف تحسين أداء تشغيل التطبيق (Dalvik Metadata). يتم إنشاؤه تلقائياً ليتوافق مع هيكل الحزم الحديثة، وهو ضروري لضمان أفضل أداء عند التثبيت الجانبي (Sideloading).
- **توليد `base.digests`**: ملف يحتوي على بصمات SHA-256 لجميع ملفات APK المختارة لضمان سلامة الملفات.
- **توليد `app.metadata`**: ملف JSON يحتوي على معلومات التطبيق الأساسية المستخرجة تلقائياً.

## هيكل الملفات المولدة:

1. **`base.dm`**: ملف ZIP يحتوي على `primary.prof` (ملف تعريف التحسين).
2. **`base.digests`**: قائمة بالملفات وبصماتها الرقمية SHA-256.
3. **`app.metadata`**: بيانات التطبيق (الاسم، الإصدار، كود الإصدار، اسم الحزمة).

## كيفية الاستخدام:

1. انسخ الكود من ملف `apk_processor.yml` المرفق.
2. ضعه في مستودعك في المسار `.github/workflows/apk_processor.yml`.
3. قم بتشغيل الـ Workflow يدوياً من تبويب Actions.

### المدخلات الرئيسية:
- **`apk_url`**: رابط تحميل ملف APKs/APKM/APKX.
- **`include_patterns`**: الملفات الداخلية المطلوب تضمينها (مثلاً: `base.apk,split_config.arm64_v8a.apk`).
- **`compression_type`**: صيغة الضغط النهائية (`zip` أو `tar.gz`).
- **`upload_destination`**: مكان الرفع (`artifacts` أو `release`).

---
**تحسينات إضافية**: تم تحسين عملية استخراج البيانات لضمان دقة معلومات التطبيق وتوافق أسماء الملفات مع جميع أنظمة التشغيل.
